## 🔹 JavaScript Classes – What Are They?

JavaScript introduced `class` syntax in **ES6 (ECMAScript 2015)** to make writing object-oriented code cleaner and more familiar (especially for developers from languages like Java, C++, etc.).

Behind the scenes, classes still use **prototypes**, but the syntax looks more structured.

---

## 🔹 Example: Basic Class in JavaScript

```javascript
class Person {
  constructor(name, age) {
    this.name = name;
    this.age = age;
  }

  greet() {
    console.log(`Hello, my name is ${this.name}`);
  }
}

const p1 = new Person("Nikhil", 24);
p1.greet(); // Output: Hello, my name is Nikhil
```

---

## 🔹 Key Features of JS Classes

| Feature          | Explanation                                                      |
| ---------------- | ---------------------------------------------------------------- |
| `constructor()`  | Special method for initializing objects                          |
| Instance methods | Defined inside the class and shared by all instances             |
| `extends`        | Used for class inheritance                                       |
| `super()`        | Calls the parent class’s constructor                             |
| `static`         | Creates a method that belongs to the class itself, not instances |

---

## 🔹 Inheritance Example

```javascript
class Animal {
  constructor(name) {
    this.name = name;
  }

  speak() {
    console.log(`${this.name} makes a noise.`);
  }
}

class Dog extends Animal {
  speak() {
    console.log(`${this.name} barks.`);
  }
}

const d = new Dog("Bruno");
d.speak(); // Bruno barks.
```

---

## 🔹 Static Methods

```javascript
class MathHelper {
  static square(x) {
    return x * x;
  }
}

console.log(MathHelper.square(5)); // 25
```

> Static methods can't be called on class instances, only on the class itself.

---

## 🔹 Classes vs Prototypes

Under the hood, classes use the **same prototype chain**:

```javascript
typeof Person; // "function"
```

> Yes, even classes are functions! `class` is just a cleaner syntax over the traditional constructor function + prototype approach.

---

## ✅ Summary

* JavaScript does have classes (`class`, `constructor`, `extends`, etc.).
* They are **syntactic sugar** over the **prototype-based inheritance**.
* Use classes when building reusable, object-oriented structures in your code.
